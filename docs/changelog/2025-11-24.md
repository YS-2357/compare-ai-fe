# 2025-11-25 변경 로그

- **상태 요약**: Streamlit 요약/에러 UX 개선, 모델별 요약만 유지하는 멀티턴 준비
- **변경 사항**:
  1. Streamlit 요약 표를 `st.table` 기반 세로 행으로 전환하고 채팅 히스토리에 표+본문을 컬럼 레이아웃으로 저장 (`app/ui/streamlit_app.py`).
  2. 에러 메시지 축약기를 적용해 부분 응답/요약/HTTP 오류 모두 detail/message만 한 줄로 표기 (`app/ui/streamlit_app.py`, `app/services/langgraph.py`).
  3. LLM 호출을 전부 try 블록 내로 이동시켜 인스턴스 생성 오류도 상태로 반환 (`app/services/langgraph.py`).
  4. 공용 히스토리 요약을 제거하고 모델별 자기 요약만 프롬프트에 포함하도록 수정 (`app/services/langgraph.py`).
- **미해결/주의**:
  - 빠른 반복 채팅 시 스트림 중단 문제는 여러 시도를 했으나 해결되지 않아 원복 상태로 유지. 다음 시도 시 `is_asking` 기반 차단, 요청 ID 필터링, 취소 이벤트 전달 방식 등을 그대로 반복하지 않도록 참고.
  - 에러 메시지 축약은 detail/message/body만 사용하므로 추가 필드가 필요하면 점검 요망.
