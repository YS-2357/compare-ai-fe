# 2025 서비스 출시 로드맵

## 1. 연간 목표
- **출시 일정**: 2025년 12월말 전 상용 서비스 공개 (웹 대시보드 + 인증 FastAPI API)
- **핵심 성과**: 일일 무료 5회 제한 + Premium 무제한 모델 비교 서비스 운영, 이메일 인증/사용량 추적/유료 플랜 전환 지원
- **준비 지표**: LangGraph 멀티 LLM 호출 안정화, 평균 응답 시간 90초 이내, 월간 활성 사용자(MAU) 5,000명 수용 가능한 인프라

## 2. 실행 전략
1. **아키텍처 분리**: FastAPI API, LangGraph 워커, Streamlit UI를 독립 서비스로 구성하고 설정/비밀 관리를 `config.py` 중심으로 통합
2. **신뢰성 있는 데이터 계층**: SQLAlchemy ORM + 사용량/히스토리 테이블로 인증·과금 기반 기능을 구현하고 마이그레이션 파이프라인 정립
3. **사용자 플로우 완성**: 가입→이메일 인증→로그인→질문→히스토리 관리→플랜 업그레이드를 단일 UX 흐름으로 묶고 Streamlit UI로 노출
4. **운영/모니터링**: LangSmith + 자체 메트릭(응답 시간/에러율/사용량)으로 가시성을 확보하고 분기별 리그레션 테스트 자동화를 구축
5. **멀티턴 멀티 LLM**: 요약 노드와 반복 Send 패턴을 이용해 “질문→LLM 병렬 호출→요약→히스토리 갱신”을 여러 턴 반복하는 파이프라인을 구축하고, 요약 모델 선택/후처리/재시도 정책을 명시

## 3. 분기별 마일스톤
| 분기 | 핵심 목표 | 주요 산출물 |
| --- | --- | --- |
| **Q1 (1~3월)** | 서비스 토대 구축 | `database.py`, `models.py`, `config.py`, Passlib/JWT 인증, LangGraph 비동기/멀티턴 구조, 기본 CI |
| **Q2 (4~6월)** | 사용량/히스토리 완성 | `usage_logs`/`query_history` 엔드포인트, Streamlit 사용량 표시, LangSmith 지표 대시보드, 캐싱/재시도 로직 |
| **Q3 (7~9월)** | 이메일 인증 및 유료 플랜 | SMTP 연동, 인증 메일, `plan` 업그레이드 API, 프리미엄 UI 차별화, 결제 연동 설계 초안 |
| **Q4 (10~12월)** | 출시 준비 및 운영 | 베타 운영, SLA 문서화, 로그/알림 파이프라인, 배포 자동화, 공식 릴리스 문서와 온보딩 자료 |

## 4. 작업 스트림 정리
- **Auth & Security**: 비밀번호 정책, JWT 만료 관리, 미인증 사용자 차단, 감사 로그
- **Usage & Billing**: 일일 제한 체크, Premium 전환, 향후 Stripe/Toss 결제 연동을 위한 Webhook 훅
- **LangGraph & Infra**: 병렬 노드 완료 보장, 재시도/타임아웃 정책, 컨테이너 기반 배포
- **Multi-turn & Summaries**: 반복 Send 플로우, 요약 모델 선택/후처리, 채팅 히스토리 저장, 인풋 병합 전략
- **Frontend/UX**: Streamlit 페이지(로그인·회원가입·인증 안내·대시보드·히스토리·설정) 완성 및 로딩 UX 개선
- **Documentation & QA**: `docs/development/` 폴더에서 발전 과정 기록, 테스트 시나리오 자동화, 운영 핸드북 작성

## 5. 리스크 및 대응
- **외부 LLM API 불안정** → 모델별 재시도/폴백 전략과 상태 대시보드 구축
- **긴 응답 시간** → 모델 선택 옵션, 스트리밍 UI, 캐시 도입
- **이메일/결제 인프라 준비 지연** → 임시 Premium 수동 업그레이드 플로우, SMTP 모킹으로 개발 병행
- **패키지 호환성** → 잠금 파일/CI 테스트 도입 및 단계별 배포 검증
